2020年2月19日にリリースされたODE0.16.1のインストール法を説明します．ここでは日本であまりメジャーではありませんが，海外ではユーザが多く私の講義でも利用しているマルチプラットフォームの開発環境Code::Blocks20.03用の記事です．

OSはLinuxのUbuntu18.04を使います。Ubuntuを使うとWindowsと同様な操作でプログラムのビルドが可能です。

  • ODE 0.16.1のインストール
    • 次のダウンロードサイトからode-0.16.1.tar.gzを保存する。ここをクリックすると直接ダウンロードする。
      • ODE ダウンロード
    • 端末を開き、以下のコマンドを実行して、保存したファイルを移動、展開する。
      • ホームディレクトリに移動
        • $ cd
      • srcディレクトリの作成
        • $ mkdir  src
      • ファイルをsrcディレクリへ移動する。~(チルダと読む)はホームディレクトに意味。つまり、/home/ユーザ名。
        • $ cp ~/Downloads/ode-0.16.1.tar.gz  ~/src
    •  次のコマンドでダウンロードしたファイルを展開する。
      • srcディレクトリへ移動
        • $ cd ~/src
      • 圧縮されたファイルを展開
        • $ tar xvzf ode-0.16.1.tar.gz
    • ビルドをするためのワークスペースを作ります。以下のようにコマンドを打ち込む．カットアンドペースト の場合は以下の文字列を使用し，自分で打ち込む場合は２個の-(マイナス)文字がつながって見えるので，下画像を参照して注意深く打ち込んでください．
      • $ cd ode-0.16.1/build
        注：cdコマンドはディレクトリの移動
      • $ sudo apt install premake4
      • premake4 --with-demos --only-double --only-static --os=linux codeblocks 
    • 成功するとcodeblocksフォルダができる．フォルダリ構成は以下となる．
      • /home/ユーザ名/src/ode-0.16.1/build/codeblocks
  • ODEのビルド
    • codeblocksを起動
      • $ codeblocks &
    • codeblocksのメニューバーから「設定」→「コンパイラ」を設定すると以下の「グローバルなコンパイラ設定」になるので、「リンカ」のタブを選び、「追加」をクリックして、以下のように"pthread"と"X11"を追加する。「OK」をクリックしてウインドウを閉じる。
    •  codeblocksのメニューバーから「ファイル(F)」→「開く（O)」を選択
      • /home/ユーザ名/src/ode-0.16.1/build/codeblocks/ode.workspaceを選択
    • 「ビルド(B)」→「ワークスペースをビルド(W)」を選択する。
      • デモプログラムの実行ファイルは/home/ユーザ名/src/ode-0.16.1/lib/Debugに生成される．
  • デモプログラムの実行
    • codeblocksの左欄にプロジェクト一覧が表示されるので実行したデモプログラムをダブルクリックして太字にする。ここでは、demo_buggyを選んだ。メニューバーの下の「実行」アイコンをクリックする。
    • 以下のバギーが現れたら成功。ごくろうさま！マウス操作で視点等を変えることができる。’a’キーを押すと前進、’z’キーで後進、’,’キーで左折、’.’キーで 右折する。なお、このバキーは３輪なのですぐ転倒してしまいますね。
