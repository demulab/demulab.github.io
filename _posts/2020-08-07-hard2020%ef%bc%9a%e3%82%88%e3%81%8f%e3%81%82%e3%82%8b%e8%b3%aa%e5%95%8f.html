２回目：ロボット制御の作り方

  • ルンバのシミュレータに関しての情報が追加されていたので、実行してみました。ビルド時にエラーが出てしまい困っております。

    • 結局のところ自分でも原因を究明することが不可であったため、ubuntuをクリーンインストールした後環境を再構築しました。結果、特に問題なくすんなりとルンバのシミュレータをインストールすることができました。
  • rvizを起動するとsegmentation faultというエラーが出て起動しません。
    • ~/.bashrcに以下の１行を追加してください。
      • export GAZEBO_IP=127.0.0.1
  • Gazeboを起動するとsegmentation faultというエラーが出て起動しません。
    • ~/.bashrcに以下の１行を追加してください。
      • export LIBGL_ALWAYS_INDIRECT=""
  • Gazeboを起動すると以下のエラーで画面が真っ暗らなままです。
    [Err] [REST.cc:205] Error in REST request
    libcurl: (51) SSL: no alternative certificate subject name matches target host name 'api.ignitionfuel.org'

    • ~/.ignition/fuel/config.yamlを次のように変更してください。
      • 変更前：url: https://api.ignitionfuel.org
      • 変更後：url: https://api.ignitionrobotics.org

 

１回目：説明会と概要

  • Windows version2004にアップデートしようとしたところ、ブルースクリーンが表示されアップデートに6回失敗しました。
    • ワークショップ標準のWSLを使うのをやめて以下のどちらかでUbuntu18.04をインストールしてください。処理が速いのでデュアルブートがお勧めですが、失敗するとWindowsの中身が全て消えるので必要なファイルのバックアップをとってから実施してください。
      • デュアルブート
        • Windows 10とUbuntu 18.04 デュアルブートする方法、パソコン工房
      • 仮想マシン
        • VirtualBox6.0+Ubuntu18.04のインストール法
  •  ワークショップにはノートPCが必ず必要ですか？
    • Raspberry Pi4でも可能です。ノートPCかRaspberry Pi4のどちらかが必要です。
  • Macでもできますか？
    • Macを持っていないのでかわりません。
  • Raspberry Pi4 RAM 2GBでは、このサイトの説明どおり実施してもインストールできません。
    • Raspberry Pi4 2GBを持っていないので対応できません。なお、ホームAIロボットのソフトウェア開発ではRAM２GBは厳しいです。新規に購入する場合は8GB版をお勧めします。
  •  WSLのUbuntuではなく、Ubuntu専用マシンでの開発環境のインストールはどうすればよいですか？
    • Ubuntu専用マシンが望ましいです。ROS MelodicのインストールはWSLと同じ方法なので以下のリンクに従ってインストールしてください。
      • Ubuntu18.04: ROS Melodicのインストール
  • NVIDIAのGPUを搭載したデスクトップPCを持っていますが、UbuntuでGPUを認識してくれません。
    • 以下のリンクに従って作業を行ってください。Xubuntuがお勧めで、NVIDIAのドライバー、CUDAも必要です。
      • CUDA 10.2のインストール
